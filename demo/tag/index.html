Overview
The payment backbone behind the Scooty app
TAG is a purpose-built SDK that gives Scooty riders PRESTO, Apple Pay, Google Pay, and Stripe — in one integration. Click through the phone to walk a real rider flow from setup to a $0.00 One Fare ride.

Step 1 — Gateway init
One call. Every service online.
TAG bootstraps the PRESTO bridge, One Fare engine, tokenization vault, and Stripe gateway in parallel — all hosted in Canadian AWS/Azure regions with zero-data VDI security.
// TAG SDK — OriginLabs import TAG from '@originlabs/tag-sdk' const gateway = await TAG.init({ apiKey: process.env.TAG_API_KEY, region: 'ca-central-1', app: 'scooty' }) // → { presto, oneFare, vault, stripe }
Canadian data residencyAuto health checks
Step 2 — Payment methods
Every method your rider expects
TAG surfaces all four payment channels through a single unified API — with PRESTO highlighted as recommended for One Fare eligibility.
const methods = await gateway.payments.list() // → ['presto', 'apple_pay', // 'google_pay', 'stripe'] await gateway.payments.add({ type: 'presto', linkMethod: 'nfc' })
Unified APITokenized storage
Step 3 — NFC PRESTO linking
Link PRESTO with a tap — no card number needed
Riders tap their physical PRESTO card to their phone. TAG validates it live against the Metrolinx API and stores a secure token. The physical card is never needed again.
const card = await gateway.presto.linkNFC() // NFC reader → Metrolinx API → token // { cardToken: 'tok_presto_••4821', // balance: 24.50, status: 'active' }
NFC readerMetrolinx validatedToken vault
Step 4 — Wallet / card
Apple Pay, Google Pay & Stripe
For non-PRESTO riders, digital wallets are registered once and reused across all rides. Stripe handles credit and debit with PCI-DSS tokenization — raw card data never touches Scooty's servers.
Zero raw card storageRecurring billing
Step 5 — Ride start
Pay as you go — no upfront fare
Because Scooty is pay-as-you-go, no destination or fare is known at ride start. TAG records the start location and selected payment method, then begins tracking. One Fare eligibility is only checked once the ride ends.
// Ride start — only location known const ride = await gateway.rides.start({ riderId: session.userId, startLocation: 'King St W, Toronto', paymentToken: session.paymentToken }) // → { rideId, startedAt, // paymentMethod, status: 'active' }
Pay as you goNo upfront fareOne Fare checked at end
Step 6 — Ride in progress
The SDK tracks the ride silently in the background
While the rider is in motion, TAG listens for ride-end events. No payment action is needed from the rider — tapping "End Ride" triggers the full fare settlement flow automatically.
// Emitted on ride end gateway.rides.on('end', async (ride) => { const result = await gateway.checkout({ riderId: ride.userId, distance: ride.km, zone: ride.transitZone }) // → One Fare check + charge })
Event-drivenZero rider input
Step 7 — Fare processing
One Fare validated in milliseconds
TAG resolves the fare end-to-end: confirms the actual ride distance and duration, then checks whether the rider's PRESTO card had a qualifying TTC or GO Transit trip within the 2-hour local or 3-hour GO transfer window. If eligible, the fare is zeroed out and the selected payment method is charged $0.00.
// One Fare check — time window only const transfer = await gateway.oneFare.check({ prestoCardToken: ride.prestoToken, rideEndTime: ride.endedAt }) // → { eligible: true, // transferType: 'TTC_LOCAL', // lastTap: '09:14 AM', // windowRemaining: '1h 12m', // finalCharge: 0.00 }
2hr local transfer3hr GO TransitTime-based, not distance
